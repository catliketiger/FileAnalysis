# 二进制文件结构化分析工具 需求规格说明书

**文档版本**：V1\.0
**适用范围**：Windows 桌面端产品全版本研发与测试
**文档目的**：明确产品的功能边界、非功能指标与迭代路径，作为研发、设计、测试团队的统一执行依据

## 一、产品概述

### 1\.1 产品定位

本产品是一款运行于 Windows 平台的专业桌面工具，面向二进制格式分析场景，通过「特征库匹配 \+ 启发式算法推断」自动识别与猜测未知结构文件的内部组织逻辑，以「十六进制视图 \+ 结构树标注」为核心交互载体，支持手动修正、补充标注与项目留存，辅助专业人员高效完成文件格式逆向、结构解析与差异分析工作。

### 1\.2 核心价值

- 降低未知二进制文件的人工逆向分析成本，自动完成基础结构初判

- 提供标准化的结构标注与编辑载体，支撑多人协作与成果沉淀

- 兼顾自动化识别效率与人工修正灵活性，适配专业场景的精细化需求

### 1\.3 目标用户

|用户角色|典型使用场景|
|---|---|
|安全逆向工程师|分析恶意样本、未知协议数据包、加密容器文件结构|
|游戏资源解包人员|逆向游戏资源包格式、解析非公开的资源打包结构|
|文档编辑器开发人员|分析非公开文档格式、验证自定义格式的二进制结构|

### 1\.4 运行环境

- 系统适配：Windows 10 及以上 64 位操作系统

- 界面语言：首版实现简体中文，预留多语言扩展框架

- 运行模式：默认本地离线运行，AI 增强识别为可选联网功能

## 二、功能需求

### 2\.1 文件管理模块

1. **文件加载能力**

    - 支持加载无扩展名、错误扩展名的纯二进制文件，以及已知格式但内部结构不公开的文件

    - 单文件支持最大处理大小为 200 MB

    - 文件加载耗时 ≤ 0\.5 秒，优先保障界面响应速度

2. **文件类型初判**

    - 基于文件名（含后缀名）进行基础文件类型判断

    - 自动区分纯文本格式文件与二进制格式文件，匹配对应默认视图

3. **异常处理规则**

    - 遇到文件损坏、加密压缩、内存不足等异常场景，直接弹出明确的异常提示

    - 不强行对异常文件做部分解析，避免误导用户

### 2\.2 多视图展示模块

#### 2\.2\.1 十六进制视图（核心基础视图）

- 支持标准十六进制内容展示，包含偏移地址、十六进制字节、ASCII 对应字符三列

- 支持字节内容的选择、复制、粘贴操作

- 内置标准十六进制工具能力：大小端字节序切换、偏移地址快速跳转、字节内容搜索

- 支持字节分组方式自定义（1/2/4/8 字节分组）

#### 2\.2\.2 纯文本视图

- 自动识别常见纯文本格式文件（txt、markdown、json、yaml、xml、csv 等），默认以纯文本形式展示

- 自动检测文件编码（UTF\-8/GBK/UTF\-16 等），支持手动切换编码，解决乱码问题

- 纯文本视图下支持内容选择、复制、手动添加备注，备注数据与十六进制视图互通

- 支持与十六进制视图无缝切换，切换后保留当前浏览位置与选中状态

#### 2\.2\.3 视图辅助能力

- **实时解码预览**：选中字节区间时，侧边栏实时展示多类型解码结果，包含数值类（uint8/int16/int32/float 等）、字符串类、时间戳类

- **书签标记**：支持对重点偏移位置添加自定义书签，实现一键快速跳转

- **多窗口拆分**：支持视图横向 / 纵向拆分，可同时查看同一文件的不同位置内容

### 2\.3 结构智能识别模块

1. **双引擎识别逻辑**

    - 引擎一：内置常见文件格式魔数特征库，基于文件头部特征做精准匹配

    - 引擎二：启发式算法推断，基于数据分布、字段长度规律、重复区块特征做结构猜测

2. **识别结果呈现**

    - 自动识别出的结构字段 / 区块，同步标注置信度（有对应计算数据时展示）

    - 识别结果以层级化结构树形式输出，对应到十六进制视图的字节区间

3. **兜底处理规则**

    - 文件完全无法被有效识别时，向用户输出引导建议，告知相关资料查询方向，引导用户手动分析

4. **性能与交互**

    - 10 MB 以内文件首次结构识别耗时 ≤ 2 秒

    - 200 MB 大文件采用异步识别模式，识别过程不阻塞前端界面操作，实时展示识别进度

5. **扩展能力**

    - 预留 AI 大模型增强识别入口，可选联网模式，对复杂未知格式做智能化结构推断

### 2\.4 结构树与视图联动模块

- 双向定位联动：点击结构树节点，十六进制视图自动跳转并高亮对应字节区间；选中十六进制字节区间时，自动高亮匹配的结构树节点

- 结构树支持折叠 / 展开层级，支持按字段名称搜索定位

- 结构树节点展示字段名称、数据类型、字节长度、偏移地址、置信度等核心信息

### 2\.5 结构手动编辑模块

1. **基础编辑能力**

    - 支持新增、修改、删除结构字段，可调整字段名称、数据类型、字节长度

    - 支持为任意字段 / 字节区间添加备注注释

2. **快捷编辑操作**

    - 框选十六进制字节区间，快速新建结构字段

    - 拖拽字段边界，实时调整字段字节长度

    - 支持多选字段，批量修改字段属性（数据类型、备注标签等）

3. **编辑保障机制**

    - 提供完整的撤销 / 重做能力，保留完整操作历史记录

    - 支持单个字段一键重置，恢复为自动识别的原始结果

### 2\.6 自定义格式规则模块

- 支持用户导入自定义格式规则库，规则文件采用 JSON / YAML 格式

- 导入规则时自动检测规则冲突，弹出冲突提示，由用户自主选择合并策略

- 自定义规则优先级高于内置特征库，匹配时优先命中用户自定义规则

### 2\.7 进阶分析模块

- **同格式文件结构对比**：支持导入两个同类型文件，执行结构差异分析

    - 支持字节级二进制差异高亮展示

    - 支持结构字段级差异比对，标注字段增减、长度变更、取值差异

    - 适用于文件版本差异分析、格式迭代对比场景

### 2\.8 项目工程管理模块

- 支持将当前分析状态保存为项目工程文件，可二次打开继续编辑

- 工程文件保存内容包含：结构标注数据、用户备注信息、原始文件元信息、原始文件哈希校验值

- 打开工程时自动校验原始文件哈希，文件变更时给出提示，保障分析一致性

### 2\.9 辅助工具模块

- 内置进制转换工具，支持二进制、十进制、十六进制相互转换

- 其余辅助工具（哈希计算、编码转换等）后续版本按需迭代

### 2\.10 系统配置模块

- **界面配置**：支持自定义界面布局、字体大小、配色主题（含深色 / 浅色模式）

- **显示配置**：支持设置默认字节分组方式、默认字节序、默认解码类型

- 所有用户配置自动持久化保存，下次启动自动生效

## 三、非功能需求

### 3\.1 性能指标

|指标项|性能要求|
|---|---|
|200MB 以内文件加载耗时|≤ 0\.5 秒|
|10MB 以内文件首次结构识别耗时|≤ 2 秒|
|200MB 文件识别模式|异步执行，界面无卡顿，支持同步进行手动操作|
|视图切换响应|纯文本与十六进制视图切换耗时 ≤ 0\.3 秒|

### 3\.2 可用性要求

- 交互逻辑符合二进制分析专业用户的使用习惯，兼容主流十六进制工具的操作快捷键

- 所有异常、错误提示清晰明确，附带问题原因与基础解决建议

- 大文件操作过程中界面保持可响应，无假死、无响应状态

### 3\.3 兼容性要求

- 兼容 Windows 10、Windows 11 全主流版本

- 适配 1080P、2K、4K 不同分辨率屏幕，支持高 DPI 缩放

### 3\.4 数据与安全

- 首版不做专项数据安全处理，默认所有分析数据仅存储在用户本地

- AI 增强识别功能为可选开启，默认关闭，联网仅传输分析所需的文件结构数据

## 四、版本迭代规划

### V0\.1 MVP 最小可用版

**核心目标**：跑通基础查看与项目留存流程，验证核心使用场景
**交付范围**：

1. 文件基础加载与后缀名类型初判

2. 十六进制基础视图（展示、选择、复制、粘贴、手动备注）

3. 纯文本格式自动识别与文本视图展示

4. 项目工程保存与打开能力

### V1\.0 核心功能正式版

**核心目标**：形成「自动识别 \+ 人工修正」的完整分析闭环
**交付范围**：

1. 双引擎结构识别能力（魔数匹配 \+ 启发式推断）、置信度标注、识别兜底提示

2. 结构树面板与十六进制视图双向联动

3. 完整的结构手动编辑能力（基础编辑 \+ 快捷操作 \+ 撤销重置）

4. 自定义格式规则库导入与冲突合并

5. 标准十六进制工具能力、实时解码预览、书签标记

6. 内置进制转换工具、基础显示配置

### V1\.1 核心功能完善版

**核心目标**：在 V1.0 基础上补齐交互细节，完善结构识别闭环
**交付范围**：

1. **扩展文件类型识别** — 增加至 140+ 种文件扩展名和 40+ 种魔数签名（含图片、音频、视频、压缩、可执行、文档、字体、虚拟机/CAD 等）
2. **预置结构规则** — 内置 18 个常见文件格式的结构特征描述（BMP、GIF、PNG、GZip、ZIP、PE/ELF/LNK、MIDI、FLAC、BZip2、TTF/OTF、ID3v2、TIFF 等），打开匹配格式文件时自动加载结构树
3. **PE 动态偏移跳转** — 自动读取 DOS Header 的 e_lfanew 值，准确定位 COFF Header 和 Optional Header
4. **打开文件自动识别** — 非文本文件打开后自动运行结构识别，无需手动点击"识别"按钮
5. **字节搜索** — 支持十六进制序列和 ASCII 文本搜索，循环遍历匹配结果
6. **框选创建字段** — 选中字节区间后右键创建结构字段
7. **结构树字段跳转高亮** — 点击树节点，十六进制视图居中并高亮对应字节区间
8. **识别进度条** — 状态栏显示识别进度指示
9. **右键快捷菜单** — 选区开始/选区结束/创建字段/添加书签
10. **复制十六进制/ASCII值** — 选中字节后 Ctrl+Shift+C 或菜单复制
11. **书签双击跳转** — 双击书签列表中条目跳转并高亮对应偏移
12. **系统设置窗口** — 工具菜单中提供分组大小、字节序、主题（浅色/深色）配置
13. **YAML 规则导入** — 规则引擎支持 YamlDotNet 解析 .yaml/.yml 格式
14. **内置规则导出** — DEBUG 模式下自动将预置规则导出为 JSON 文件到 Rules 目录
15. **纯文本跳过识别** — 纯文本文件打开后自动切换到文本视图，跳过结构识别
16. **文件关闭前清空视图** — 打开新文件前自动关闭旧文件并清空十六进制视图、文本视图和结构树
17. **右键菜单静态化** — 修复右键菜单闪烁和重复订阅问题，添加书签、创建字段操作稳定

**测试覆盖**：87 个单元/集成测试，覆盖核心模型、识别引擎、编辑系统、规则引擎

## 五、边界与约束

1. 产品暂不提供命令行调用能力，不支持脚本化批量自动化处理

2. 单文件官方支持上限为 200 MB，超大小文件不做兼容性保障

3. 自动结构识别为猜测性结果，不保证 100% 准确率，仅作为人工分析的辅助参考

4. 首版仅提供简体中文界面，多语言能力后续迭代

5. 首版不做专项数据安全与脱敏处理，敏感文件分析由用户自行保障数据安全
