# FileStruct 功能增强需求规格说明书

**文档版本**：V1.0
**适用范围**：V1.X 后续迭代

---

## 功能一：压缩类格式文件结构展开

### 1.1 概述

支持对本质为 ZIP 压缩包的文档/电子书格式（DOCX/PPTX/XLSX/EPUB/ODF/ZIP 等）进行内部结构展开，显示压缩包内文件目录。

### 1.2 行为规范

| 项目 | 规格 |
|------|------|
| 触发方式 | 自动识别 → 结构树显示根节点 → 用户右键 → 「展开...」 |
| 展开深度 | **一层**：仅显示 ZIP 内文件条目，不进一步展开条目内部 |
| 条目信息 | 文件名、在 ZIP 中的偏移、原始大小、压缩后大小 |
| 条目点击 | HexView 跳转到该条目数据所在偏移位置 |
| XML 处理 | **不**格式化解析 XML 内容 |

### 1.3 支持的格式

| 扩展名 | 格式名称 | 识别方式 |
|--------|----------|----------|
| .docx/.pptx/.xlsx | Office Open XML | ZIP 魔数 + 扩展名 |
| .epub | EPUB 电子书 | ZIP 魔数 + 扩展名 |
| .odt/.ods/.odp | OpenDocument | ZIP 魔数 + 扩展名 |
| .zip/.jar/.apk | 标准 ZIP | ZIP 魔数 |

### 1.4 用户交互

```
结构树:
├─ ZIP 压缩包 [已识别]
│  右键 → 「展开压缩包」
│  ├─ [Content_Types].xml    @ 0x0000  1.2 KB
│  ├─ docProps/core.xml      @ 0x0500  0.8 KB
│  ├─ word/document.xml      @ 0x0800  45 KB
│  ├─ word/media/image1.png  @ 0xB400  256 KB
│  └─ ...
```

## 功能二：可执行文件压缩格式识别

### 2.1 概述

识别 EXE/DLL 文件是否被加壳压缩（UPX/ASPack/Petite 等），在结构树中标记对应的 Section 名称和描述。

### 2.2 行为规范

| 项目 | 规格 |
|------|------|
| 识别深度 | **仅识别+标记**：显示加壳类型和 Section 信息，不尝试解压缩 |
| 触发方式 | 打开 EXE/DLL → 自动识别 PE 结构 + 壳特征 |
| Section 标记 | 结构树中标注 Section 名称、虚拟地址、原始大小等，壳特征作为 Section 描述 |

### 2.3 支持的加壳格式（首期）

| 加壳器 | 特征 |
|--------|------|
| UPX | Section 名 `.upx0`/`.upx1`，入口点特征 |
| ASPack | Section 名 `.aspack`/`.adata` |
| Petite | Section 名 `.petite` |
| 其他 | 后续迭代扩展 |

### 2.4 结构树展示

```
PE
├─ DOS Header
├─ COFF Header
├─ Optional Header
├─ Section: .upx0 (UPX 压缩代码段)
├─ Section: .upx1 (UPX 压缩数据段)
├─ Section: .rsrc (资源段)
```

## 功能三：文件内嵌资源提取

### 3.1 概述

用户右键结构树节点 →「扫描内嵌资源」→ 在选中节点数据范围内用魔数签名扫描已知格式 → 发现后添加虚拟子节点 → 对资源节点右键 →「导出资源」

### 3.2 行为规范

| 项目 | 规格 |
|------|------|
| 触发方式 | 右键结构树节点 →「扫描内嵌资源」 |
| 扫描范围 | 仅限选中节点的偏移范围，不全局扫描 |
| 扫描方法 | 用现有 BuiltinSignatures 魔数库在数据范围内匹配 |
| 扫描终止 | 找到即添加虚拟节点，继续向后扫描，限制最多 200 个发现 |
| 导出 | 右键资源节点 →「导出资源...」→ 保存为独立文件（用数据范围 + 魔数对应的扩展名） |

### 3.3 支持的资源类型

| 类别 | 格式 | 导出扩展名 |
|------|------|-----------|
| 图片 | PNG/JPEG/GIF/BMP/ICO | .png/.jpg/.gif/.bmp/.ico |
| 音频 | WAV/MP3/OGG/FLAC/MIDI/AIFF | .wav/.mp3/.ogg/.flac/.mid/.aiff |
| 视频 | MP4/MKV | .mp4/.mkv |
| 压缩包 | ZIP/RAR/GZip/Zstd/7z/CAB | .zip/.rar/.gz/.zst/.7z/.cab |
| 可执行 | PE/ELF | .exe/.elf |
| 字体 | TTF/OTF/WOFF/WOFF2 | .ttf/.otf/.woff/.woff2 |

### 3.4 用户交互

```
结构树:
├─ data.bin [128 KB]
│  右键 → 「扫描内嵌资源」
│  ├─ PNG Image    @ 0x1000  32 KB  [导出资源...]
│  ├─ PE Executable @ 0x9000  64 KB  [导出资源...]
│  └─ WAV Audio    @ 0x1A000 16 KB  [导出资源...]
```

## 实施优先级

| 序号 | 功能 | 优先级 | 预计复杂度 |
|------|------|--------|-----------|
| 1 | 压缩类格式结构展开 | 最高 | 中（ZIP 目录解析） |
| 2 | 可执行文件压缩识别 | 中 | 低（新增 PE Section 特征） |
| 3 | 内嵌资源扫描提取 | 低 | 中（魔数扫描 + 导出逻辑） |

---

## 版本兼容性

- 不修改现有 `.fstruct` 项目文件格式
- 内嵌资源「虚拟节点」使用 `Source = AutoDetected`，`Confidence = 0.7`
- 压缩包展开的 ZIP 条目节点使用 `Source = AutoDetected`
